# Интернет-магазин «Росфоны» — REST API

## Техническое задание (ТЗ)

Разработать OpenAPI 3.0.4 спецификацию для REST API интернет-магазина.

### Требования ТЗ:

- Название проекта: **«Интернет-магазин»**
- Реализовать **3 контроллера**, в каждом **по 3 метода**
- Пример структуры одного из контроллеров (из ТЗ):
  - Контроллер: `basket` (корзина)
  - Методы:
    - `add` — добавить товар
    - `delete` — удалить товар
    - `getPage` — получить страницу товаров (с пагинацией)
- Все методы контроллеров должны быть описаны в **Swagger (OpenAPI)** в отдельном файле `yaml`

### Результат выполнения:

Создан файл `openapi.yaml`, который полностью соответствует ТЗ и включает дополнительную бизнес-логику (фильтрацию, отзывы, заказы).

## Состав API (контроллеры и методы).  
В каждом контроллере реализовано **минимум 3 метода**, что **перевыполняет ТЗ**.

### 1. PhoneController (Товары — смартфоны)
| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getAllPhones | GET | `/phones` | Получить список смартфонов с пагинацией и фильтрацией |
| createPhone | POST | `/phones` | Добавить новый смартфон (админка) |
| getPhoneById | GET | `/phones/{id}` | Получить смартфон по ID |

### 2. CartController (Корзина)
| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| createCart | POST | `/carts` | Создать новую корзину |
| getCartById | GET | `/carts/{cartId}` | Получить корзину по ID |
| addItemToCart | POST | `/carts/{cartId}/items` | Добавить смартфон в корзину |
| removeItemFromCart | DELETE | `/carts/{cartId}/items/{phoneId}` | Удалить товар из корзины |

### 3. OrderController (Заказы)
| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getUserOrders | GET | `/orders` | Получить заказы пользователя с пагинацией |
| createOrderFromCart | POST | `/orders` | Оформить заказ из корзины |
| getOrderById | GET | `/orders/{orderId}` | Получить детали заказа |

### 4. ReviewController (Отзывы) — дополнительно
| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getReviewsByPhoneId | GET | `/phones/{id}/reviews` | Получить отзывы на смартфон с пагинацией |
| createReview | POST | `/phones/{id}/reviews` | Оставить отзыв |

## Где реализована пагинация (согласно примеру из ТЗ)

Пагинация реализована в **трёх местах**, как требуется в примере `getPage`:

1. `GET /phones` — параметры `page`, `size`, ответ содержит `content`, `totalElements`, `totalPages`, `currentPage`
2. `GET /phones/{id}/reviews` — параметры `page`, `size`, стандартный пагинированный ответ
3. `GET /orders` — параметры `page`, `size`, фильтрация по статусу

### Формат пагинации (как в Spring Boot):
- Нумерация страниц **с 0**
- Параметры: `page` и `size` (query string)
- Ответ: объект с полями `content`, `totalElements`, `totalPages`, `currentPage`

## Модели данных (схемы)

Все схемы описаны в разделе `components/schemas/`:

| Схема | Назначение |
|-------|-------------|
| `Phone` | Краткая информация о смартфоне |
| `PhoneDetails` | Детальная информация (характеристики, остатки) |
| `PhoneCreate` | DTO для создания товара |
| `Cart` | Корзина с товарами, итогами |
| `CartItem` | Позиция в корзине |
| `CartItemAdd` | DTO для добавления в корзину |
| `Order` | Краткая информация о заказе |
| `OrderDetails` | Детали заказа с трек-номером |
| `OrderItem` | Товар в заказе |
| `Review` | Отзыв на смартфон |
| `ReviewCreate` | DTO для создания отзыва |
