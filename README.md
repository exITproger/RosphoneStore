# Интернет-магазин «Росфоны» — REST API

## Техническое задание (ТЗ)

Разработать OpenAPI 3.0.4 спецификацию для REST API интернет-магазина.

### Требования ТЗ:

- Название проекта: **«Интернет-магазин»**
- Реализовать **3 контроллера**, в каждом **по 3 метода**
- Пример структуры одного из контроллеров (из ТЗ):
  - Контроллер: `basket` (корзина)
  - Методы:
    - `add` — добавить товар
    - `delete` — удалить товар (очистка корзины)
    - `getPage` — получить страницу товаров (с пагинацией)
- Все методы контроллеров должны быть описаны в **Swagger (OpenAPI)** в отдельном файле `yaml`

### Результат выполнения:

Создан файл `openapi.yaml`, который полностью соответствует ТЗ.

## Состав API (контроллеры и методы)

В каждом контроллере реализовано **ровно 3 метода**, что полностью соответствует ТЗ.

### 1. PhoneController (Товары — смартфоны)

| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getAllPhones | GET | `/phones` | Получить список смартфонов с пагинацией |
| createPhone | POST | `/phones` | Добавить новый смартфон (админка) |
| getPhoneById | GET | `/phones/{id}` | Получить смартфон по ID |

### 2. CartController (Корзина)

| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getCartById | GET | `/carts/{cartId}` | Получить корзину по ID |
| addItemToCart | POST | `/carts/{cartId}/items` | Добавить смартфон в корзину (`add`) |
| clearCart | DELETE | `/carts/{cartId}/items` | Очистить корзину (`delete`) |

> Примечание: метод `getPage` для корзины не требуется, так как пагинация реализована в каталоге товаров (`GET /phones`).

### 3. OrderController (Заказы)

| Метод | HTTP | Эндпоинт | Описание |
|-------|------|----------|----------|
| getUserOrders | GET | `/orders` | Получить заказы пользователя с пагинацией (`getPage`) |
| createOrderFromCart | POST | `/orders` | Оформить заказ из корзины |
| getOrderById | GET | `/orders/{orderId}` | Получить детали заказа |

## Где реализована пагинация (согласно примеру из ТЗ)

Пагинация реализована в **трёх местах**:

| Эндпоинт | Параметры пагинации |
|----------|---------------------|
| `GET /phones` | `page`, `size` |
| `GET /orders` | `page`, `size` |

### Формат пагинации:
- Нумерация страниц **с 0**
- Параметры: `page` и `size` (query string)
- Ответ: объект с полями `content`, `totalElements`, `totalPages`, `currentPage`, `pageSize`

### Пример ответа `GET /phones?page=0&size=12`:

```json
{
  "content": [
    { "id": 101, "model": "R5 Pro", "price": 34990 },
    { "id": 102, "model": "X10", "price": 24990 }
  ],
  "totalElements": 156,
  "totalPages": 13,
  "currentPage": 0,
  "pageSize": 12
}
